Architecture

PII stays inside the bank. Signals cross the boundary.

A single container in your private network is the only thing that touches raw customer data. Everything downstream sees pseudonyms.

Raw PII — stays inside your network Pseudonymized signal — HMAC-signed Durable spool — survives Hub outages
Transformation

PII in, pseudonymized signal out

Deterministic, keyed transforms. Identity becomes a mosaic; amount becomes a tier; location becomes a zone; time becomes a bucket.

// raw transaction (in the bank)
{
  "national_id": "22345678901",
  "name": "John Doe",
  "account": "ACC-1234567890",
  "amount": 9500.00,
  "latitude": 6.4541,
  "longitude": 3.3947,
  "timestamp": "2026-01-15T14:07:33Z"
}
anonymize + sign 
// signal sent to the Hub
{
  "identity_mosaic": "d433f5d2…",
  "mosaic_scope": "global",
  "mosaic_version": 2,
  "timestamp": "2026-01-15T14:00:00Z",
  "metadata": {
    "amount_tier": "TIER_3",
    "location_zone": "s14kt",
    "is_near_threshold": true
  }
}
Capabilities

Built for cross-bank detection, without the exposure

🔗

Cross-bank identity mosaics (v2)

With a national identifier, the identity mosaic is an HMAC-SHA256 keyed only by the shared regional pepper — so the same person yields the same mosaic at every member bank, enabling cross-institutional pattern detection. Without one, it falls back to a bank-local mosaic. Every signal is tagged global or local so the Hub knows what it may correlate.

💾

Durable spool

Signals are persisted before acknowledgment and delivered by a background forwarder. Hub outages and restarts don't lose data — permanently rejected signals dead-letter instead of blocking the queue.

🎭

Privacy-preserving transforms

Amount → tier, lat/long → geohash zone, timestamp → 15-minute UTC bucket, account / device / IP → salted hashes. Enough signal to detect fraud, not enough to re-identify.

🕵️

Structuring & mule-route flags

Near-threshold amounts are flagged for multi-bank structuring; a counterparty's destination mosaic matches their own identity mosaic, so the Hub can follow mule routes across institutions.

🔐

Signed & authenticated

HMAC-SHA256 payload signing to the Hub, inbound API-key auth on /process, optional TLS, a 1 MB body cap, and PII-free structured logs.

📦

One hardened binary

Distroless, nonroot, no shell — a single static Go binary with no runtime dependencies. Graceful shutdown, bounded retry with backoff, and a self-probing health check.

One gateway, many detections

Match the same fraudster across member banks by global identity mosaic — without any institution exposing customer identities to another.

Get started

Running in minutes, not months

1

Clone & configure

Copy .env.example and set your BANK_SALT, REGIONAL_PEPPER, Hub credentials, and HMAC_SECRET.

2

Deploy the container

Run the distroless image inside your private network. Set INBOUND_API_KEY and TLS, point SPOOL_DIR at a volume.

3

POST your transactions

Send raw transactions to /process. The gateway anonymizes, signs, and forwards to the Hub — and returns 202.

# run the gateway
$ docker run -p 8080:8080 \
    -e INSTITUTION_ID=BNK_LAGOS \
    -e BANK_SALT=$SALT \
    -e REGIONAL_PEPPER=$PEPPER \
    -e HMAC_SECRET=$SECRET \
    -e SPOOL_DIR=/spool \
    edge-gateway

# send a transaction
$ curl -X POST localhost:8080/process \
    -H "Authorization: Bearer $KEY" \
    -d '{ "national_id":"2234…", "amount":9500, … }'
